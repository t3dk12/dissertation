# Dissertation

This repository is my Part C lithium project workspace. It started from I-ReaxFF, was cleaned for this project, and now runs with the restored Desktop `I-ReaxFF/irff` loading path that is currently working for my workflow.

## Project Goal

- Train a lithium-focused IRFF/MPNN model from OUTCAR-derived trajectories.
- Keep the workflow reproducible: setup -> optional dataset build -> training -> conversion/check.
- Keep the repo tidy enough to run easily on the remote Linux machines.

## Current Working State

- The active IRFF core in `I-ReaxFF/irff/` now matches the working Desktop copy used for this project.
- `run_scripts/train_without_force_Li_NN.py` supports dataset filtering with:
  - `all`
  - `bcc`
  - `fcc`
  - `bcc_fcc` (also accepts `both` and `bcc+fcc`)
- The simplest current Linux training entrypoint is:

```bash
bash run_bcc.sh
```

This runs BCC-only training through the standard `train.sh` flow.

## How This Differs from Base I-ReaxFF

This repository started from a pure clone of:

`https://github.com/fenggo/I-ReaxFF.git`

Then it was updated for this project by:

1. Adding project run scripts and setup scripts.
2. Adding raw standard force fields (`44_36`, `179_3`, `2024vD`).
3. Pruning non-essential upstream folders from `I-ReaxFF` (`doc`, `examples`, `test`, most `tools`).
4. Adding the conversion helpers used in this workflow.
5. Keeping the active run potential at root `ffield.json`.
6. Restoring the working Desktop `I-ReaxFF/irff` loader path after the quarantine-based loader proved unreliable for the full lithium dataset.

## Repository Layout

- `I-ReaxFF/` editable project copy used for this workflow.
- `run_scripts/` project scripts:
  - `build_dataset_duplicate.py`
  - `train_without_force_Li_NN.py`
- `setup.sh` environment and dependency setup (Python 3.10, Conda env `li-model`).
- `train.sh` generic training entrypoint.
- `run_bcc.sh` one-command BCC training launcher for the Linux machines.
- `setup_and_run.sh` setup + optional dataset build + training runner.
- `ffields/raw/` source text potentials.
- `ffields/full_json/` full-element converted potentials.
- `ffields/li_only/` lithium-only converted potentials.
- `ffields/reference/` reference JSON files used in the conversion flow.
- `verify/` conversion verification outputs.
- `data/Li-metal_OUTCARs/` placeholder location for the OneDrive OUTCAR data.

## Data Note

OUTCAR files are too large for Git and are not stored in this repository.

Download the OUTCAR dataset from OneDrive and place it in:

`data/Li-metal_OUTCARs/`

Trajectory files are generated by the dataset script into `data/traj/`.

If `data/traj/` already exists on the Linux machine, you do not need the OUTCAR folders just to train.

## Linux Quick Start

Run from repository root on the Linux machine.

### Normal BCC Run (Recommended)

```bash
cd /home/lunet/mpek/dissertation
git pull origin main
bash run_bcc.sh
```

This assumes:

- the repo is already cloned
- the `li-model` conda environment already exists
- the dataset already exists in `data/traj/`

### Generic Training Options

- BCC only:

```bash
TRAIN_DATASET_GROUP=bcc bash train.sh
```

- FCC only:

```bash
TRAIN_DATASET_GROUP=fcc bash train.sh
```

- BCC + FCC:

```bash
TRAIN_DATASET_GROUP=bcc_fcc bash train.sh
```

## Setup / Build / Train Split

### 1) Initial Setup

Run this once per machine, or again if the environment needs repairing.

```bash
bash setup.sh
```

### 2) Optional Dataset Build

Only needed when rebuilding `data/traj/` from OUTCAR files.

```bash
python run_scripts/build_dataset_duplicate.py \
  --input_dir data/Li-metal_OUTCARs \
  --output_dir data/traj \
  --duplicate 1 \
  --ffield ffield.json \
  --cutoff 5.0
```

Expected outputs:

- `data/traj/train/*.traj`
- `data/traj/test/*.traj`
- `data/traj/dft_summary.csv`

### 3) Normal Training

Generic command:

```bash
python run_scripts/train_without_force_Li_NN.py \
  --train_dir data/traj/train \
  --ffield ffield.json
```

Example full command used in this workflow:

```bash
python run_scripts/train_without_force_Li_NN.py \
  --train_dir data/traj/train \
  --ffield ffield.json \
  --dataset_group bcc \
  --s 10000 --lr 0.0001 --pr 200 --writelib 1000 --batch 50 \
  --bo 0 --h 0 --a 0 --t 0 --f 0 --vdw 1
```

## One-Command Scripts

- `bash run_bcc.sh`
  - current recommended Linux command
  - runs BCC-only training
  - does not rebuild the dataset
  - does not rerun setup

- `bash train.sh`
  - generic training script
  - can be combined with `TRAIN_DATASET_GROUP=bcc|fcc|bcc_fcc`

- `bash setup_and_run.sh`
  - full setup + optional dataset build + training path
  - useful for a fresh machine, but not the normal remote Linux workflow once the machine is prepared

## Conversion / Verification Commands

```bash
python I-ReaxFF/tools/ffield_to_json_full.py -f ffields/raw/2024vD_FF.txt -o ffields/full_json/2024vD_ffield.json

python I-ReaxFF/tools/json_to_ffield.py
```
